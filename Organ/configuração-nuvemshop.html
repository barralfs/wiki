Realizando o Login na Plataforma

Passo 1: Acessamos a tela de 'Empresas', dentro de 'Configurações do Sistema'.

Passo 2: Clicar em 'Consultar ID', conforme a imagem acima.

Passo 3: Seremos direcionados ao site e nele vamos realizar o login, com as informações obtidas com o responsável pelo NuvemShop.

Passo 4: Vamos nos certificar de que o ID da loja foi devidamente carregado na tela de 'Empresas'.

Ajustando o Cadastro dos Produtos

Passo 1: Abrir o módulo de 'Produtos e Serviços', dentro de 'Cadastros'. 

Passo 2: Conferir se o campo 'Tipo Produto' está preenchido como 00 - Mercadoria Para Revenda ou 04 - Produto Acabado.

Passo 3: Conferir se o campo 'Sincronizar Marketplace' está marcado.

Passo 4: Informamos o 'Peso Bruto' do produto, dentro de 'Geral', da aba de 'Impostos'.

Observação: O preenchimento do campo é extremamente importante, pois é levado em conta no cálculo de frete.

Passo 5: Na aba de 'Outras Informações', realizaremos o preenchimento dos seguintes campos.

Título da SEO: Será o nome do produto exibido na plataforma.

Descrição da SEO: Descrição simplificada do produto, deve ser objetiva. Os caracteres preenchidos aqui podem ativar buscas no Google.

Descrição Completa: Especificação completa das informações do produto, como a sua composição, dimensões, peso e observações em geral.

Estruturando as Fotos dos Produtos

Passo 1: Criar a pasta 'Fotos', na raiz do Organ.

Passo 2: Vamos inserir novas pastas, dentro de 'Fotos', que vão corresponder ao 'Código do Produto'.

Passo 3: Dentro das pastas com o código, inserimos os arquivos de imagem. Eles deverão ser nomeados com uma numeração, conforme exemplificado abaixo. A numeração corresponderá à relevância das fotos, portanto a foto '01', será exibida primeiro e assim sucessivamente.

Configurando os Preços, Local de Estoque e Quantidades

Passo 1: Identificar a tabela 'CONFIG_SQL' no banco de dados e criar a sentença 'NUVEMSHOP_PRODUTO'.

Passo 2: Conforme a necessidade do cliente, podemos gerar um select que retorne as informações desejadas, como preço, empresa, local de estoque e quantidades a serem enviadas. 

Segue modelo de sql com a consulta realizada, vale ressaltar que este não é um padrão oficial, é apenas um exemplo para se ter uma ideia. CUIDADO! Existem informações específicas da regra de negócio do cliente.

SELECT
  CAST(:ID_EMPRESA AS INTEGER) AS ID_EMPRESA,
  PD.ID_PRODUTO,
  PD.ID_CATEGORIA_PRODUTO,
  PD.ID_SUBCATEGORIA_PRODUTO,
  MP.DESCRICAO AS MARCA,
  PD.DESCRICAO,
  SEO_TITULO,
  SEO_DESCRICAO,
  MARKETPLACE_DESCRICAO,
  PD.PRECO AS PRECO_PRODUTO,
  PD.PESO_BRUTO,
  PD.LARGURA,
  PD.ALTURA,
  PD.PROFUNDIDADE,
  PD.CONTROLAR_ESTOQUE,
  COALESCE((SELECT FIRST 1 CODIGO FROM CODIGO_BARRAS_PRODUTO WHERE ID_PRODUTO = PD.ID_PRODUTO ORDER BY CHAR_LENGTH(CODIGO)), CODIGO) AS COD_BARRAS,
  (SELECT SALDO_ESTOQUE FROM SP_ESTOQUE_SALDO(:ID_EMPRESA, PD.ID_PRODUTO, 0, CURRENT_DATE, 1)) AS QUANT_ESTOQUE_ATUAL,
  CASE
    WHEN (INATIVO = 'S') THEN 'S'
    WHEN (SINCRONIZAR_MARKETPLACE = 'N') THEN 'S'
    WHEN (TIPO_PRODUTO NOT IN (0, 4)) THEN 'S'
    ELSE 'N'
  END AS INATIVO,
  IP.IMAGEM
FROM PRODUTO PD
LEFT OUTER JOIN IMAGEM_PRODUTO IP USING(ID_PRODUTO)
LEFT OUTER JOIN MARCA_PRODUTO MP USING (ID_MARCA_PRODUTO)
WHERE ((PD.ID_PRODUTO = :ID_PRODUTO) OR (:ID_PRODUTO = 0)) AND (TIPO_PRODUTO IN (0, 4))
